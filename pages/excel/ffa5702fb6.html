VDB

VDB 函数基于指定的折旧率得出所选时间间隔内的资产的折旧额。

VDB(成本, 残值, 折旧年限, 首期, 末期, 折旧因子, 不切换)

成本： 要折旧的资产的成本。成本是数字值，通常格式化为货币且必须大于或等于 0。成本通常是购买价格，包括税金、运费和安装费。某些税收利益可从成本中扣除。

残值： 资产的折余价值。残值是数字值，通常格式化为货币且必须大于或等于 0。

折旧年限： 数字值，表示资产折旧期数，称为折旧年限或预计使用年限。折旧年限必须大于 0。允许折旧年限的小数部分（例如，5.5 表示五年半的折旧年限）。

首期： 表示要包括在计算中的第一期的数字值。首期必须大于 0 且小于末期。

末期： 表示要包括在计算中的最后一期的数字值。末期必须大于 0 且大于首期。

折旧因子： 可选数字值，确定折旧率。折旧因子指定为小数或百分比（使用百分比符号）。如果忽略了，则假定为 2（200% 表示双倍递减）。数字越大，折旧就越快。例如，如果需要折旧率为直线折旧的 1.5 倍，则使用 1.5 或 150%。

不切换： 可选模态值，指示折旧是否切换到直线方法。

切换(0、FALSE 或省略)： 在直线折旧超过递减余额折旧的年份切换到直线方法。

不切换(1、TRUE)： 请勿切换到直线方法。

注释

  • VDB 函数基于可变余额递减法得出折旧。

  • 首期应指定为您希望包括在计算中的第一个周期之前的周期。如果您希望包括第一个周期，请使用 0 表示首期。

  • 如果您希望确定仅包括第一个周期的折旧，末期应该为 1。

  • 此函数结果中显示的货币取决于“语言与地区”设置（在 macOS 的“系统偏好设置”和 iOS 的“设置”中），或者 iCloud 设置中的“时区 / 地区”设置。

示例

假定所购买资产的成本为 ￥11,000.00（成本），在折旧年限达到约 5 年（折旧年限）后，估计折余价值为 ￥1,000（残值）。您计划使用 1.5 (150%) 递减余额法（折旧因子）来折旧资产。

=VDB(11000, 1000, 5, 0,1, 1.5, 0) 得出 ￥3,300，此值是第一年（首期为 0，末期为 1）的折旧。

=VDB(11000, 1000, 5, 4, 5, 1.5, 0) 得出 ￥1,386.50，此值是第五年（最后一年）的折旧，这里假定当此值大于递减余额折旧时采用直线折旧（不切换为 0 或 FALSE）。

=VDB(11000, 1000, 5, 4, 5, 1.5, 1) 得出 ￥792.33，此值是第五年（最后一年）的折旧，这里假定一直采用递减余额折旧（不切换为 1 或 TRUE）。
